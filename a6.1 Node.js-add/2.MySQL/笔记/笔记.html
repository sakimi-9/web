MySQL与MongoDB 区别
    MySQL 和 MongoDB 是两种不同类型的数据库管理系统，主要区别在于它们的数据存储方式和查询语言：
  1. MySQL是一个关系型数据库，使用结构化查询语言（SQL）进行数据操作。它将数据存储在表格中，每一行对应一条记录，每一列对应一种属性。MySQL 支持事务处理，并且可以很好地处理复杂查询和多表连接。MySQL通常通过垂直扩展（增加单一服务器的能力）来提升性能。 。

  2. MongoDB是一个非关系型（NoSQL）数据库，它以文档的形式存储数据，通常这些文档是 JSON 格式的 BSON 文档。MongoDB 不需要预先定义表格结构，并且易于扩展，适合处理大量非结构化数据。MongoDB设计之初就考虑到了水平扩展（增加更多的服务器来分散负载），适合构建分布式系统

  3. 简而言之： MySQL：关系型，基于表格，使用 SQL 查询。 MongoDB：非关系型，基于文档，使用 BSON 存储数据。
