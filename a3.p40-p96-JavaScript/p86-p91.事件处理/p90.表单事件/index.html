事件类型之表单事件

input 输入框输入时触发

格式:Dom0级 input节点.oninput = function（）{事件处理}
select 在输入框中选中内容时触发

格式:Dom0级 input节点.onselect = function（）{事件处理}
change 输入框内容改变完成时触发
改变完成，敲回车或者失去鼠标焦点（鼠标点击文本框之外的区域） | change的触发频率要比input低很多

格式:Dom0级 input节点.onchange = function（）{事件处理}

这三个都是与输入框强相关，都是写在input节点后
reset() 这是一个表单节点的方法 表单重置（清空）时(表单成员变回默认值)触发
点击按钮触发，清空表单 | 所以是写在button节点的onclick的监听函数中，调用表单方法的形式出现

格式:Dom0级 button节点.onclick = function（）{表单变量名.reset（）;事件处理}
submit 当表单数据向服务器提交时触发
在input标签里设置好name="文本框名" 到时候提交数据时 会以 文本框名=内容 提交，意思是 名为文本框名的input里的内容为"内容"

格式:HTMl级也可以 form标签里 onsubmit="函数（）" js : function 函数名（）{事件处理}

Dom0级 form节点.onsubmit = function（）{事件处理}
