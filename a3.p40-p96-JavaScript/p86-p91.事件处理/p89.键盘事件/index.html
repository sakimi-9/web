事件类型之键盘事件

键盘事件类型
格式:Dom0级时 元素节点.on事件类型 = function（）{事件处理}
keydown 按键按下触发
现在与键盘按键的输入也是同步触发
keyup 按键抬起触发
与键盘按键的输入同步触发，键盘事件类型中一般用这个类型
keypress 按下有值按键触发，无值按键不会触发
无值按键有ctrl del shift F1 等等 ，有值按键是字符（数字 字母 空格）按键
有keydown时，优先触发keydown

与键盘事件相关的 Event 属性 keyCode
keyCode 返回按下按键的唯一标识（数字）
如何使用 放在键盘事件的监听函数中
比如:打印按键的唯一标识 元素节点.on键盘事件类型 = function（e）{console.log(event.keyCode);事件处理}
最常用的一个按键标识 回车的唯一标识为13 空格的唯一标识为32
回车的唯一标识用处 常用于检测按下回车后进行某些事件（如 开始搜索 然后跳转 ）

格式:在键盘事件的监听函数中嵌套 if（e.keyCode === 13）{按下回车的事件处理}


与键盘事件相关的 Event 属性 target 中的 value
  • 常用于返回用户输入的内容 使用:在键盘事件的监听函数中 event.target.value
  • value(值)
    1. 在表单的input标签里 value="代替文本"
    2. 在setAttributeNode里 id/class.value = "id名/类名"
    3. 在表单form的event的targer里 e.target.value 代表用户输入的内容
