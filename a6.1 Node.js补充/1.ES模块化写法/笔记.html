CommonJS与ES Modules 的叫法 这两都是js文件模块化的一种规范

  • cjs模块化写法 require导入方法 都-> CommonJs

  • ES/ESM/mjs模块化写法 import/expor的导入导出方法 都-> ES Modules
要在Nodejs文件中使用 ES模块化写法 则要告诉 Node.js 你的文件是一个 ES 模块，可以在文件扩展名使用 .mjs 或者在 package.json 文件中设置 "type": "module"

还要注意 ES模块化写法 导入的路径 要写完整(包括文件扩展名),CommonJS的 写不写完整都行
一个项目中不能同时使用两种模块化写法
CommonJS与ES Modules 的区别
  • 导入/导出语法：CJS 使用 require 和 module.exports，而 ESM 使用 import 和 export。
  • 加载方式：CJS 是同步的，而 ESM 是异步的。
  • 运行时 vs 编译时：CJS 在运行时解析模块依赖，ESM 在编译时解析。
  • 互操作性：虽然 Node.js 支持两种模块系统，但在某些情况下，它们可能无法直接互操作
  • 兼容性:CommonJs的对导入外部依赖的兼容性更高,毕竟Nodejs以这种起家的
